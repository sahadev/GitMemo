Desktop and Android · Design System v1.0

GitMemo 软件本身应像一个安静、耐用、密集但不拥挤的知识工作台。

官网负责场景叙事；软件负责日常捕获、搜索、阅读、编辑、同步和归档。应用内部不要做营销式 hero，也不要用装饰性视觉抢占工作空间。

Branch Context

应用内容区优先服务阅读与编辑。Markdown 视图使用系统比例字体，路径、元数据、按钮与导航保持等宽字体。

代码、路径和命令使用 JetBrains Mono，内容可选择、可复制、可导出。

排版规则

应用外壳

侧边栏、列表、按钮、路径、设置和元数据使用 JetBrains Mono，保持开发者工具的精确感。

阅读表面

Markdown 阅读和 PDF 导出使用系统比例字体。长文阅读优先，行高约 1.72。

字号

应用内部标题控制在 16-20px；列表标题 13-15px；元数据 10-11px。不要使用官网式大标题。

布局原则

Desktop固定左侧栏，内容使用列表/详情分栏；列表宽度可调整；详情区保持稳定工具栏。
Mobile使用底部导航和列表/详情栈，不把桌面分栏硬塞到手机上。
页面密度桌面内容 padding 为 20px 28px；移动端为 14px；固定格式控件要有稳定尺寸。

组件规范

Sidebar

200px 宽；图标 16px；导航行 10px 16px；激活态使用 hover 面和 accent 文本。

Toolbar

紧凑、稳定、路径可截断。常用动作用图标按钮：返回、编辑、保存、收藏、导出、更多。

Cards

只用于统计、设置组、导入结果、搜索结果等真实分组。半径 6-8px，无重阴影。

Lists

整行可点击，用 border-bottom 分隔。标题单行截断，预览最多两行，元数据始终可见。

Markdown

内容可选择。代码块用背景、边框和 6px 半径；图片最大宽度 100%，半径 4px。

PDF

导出时使用白底、深色文字、系统字体、12pt 正文和约 760px 最大宽度，不带应用外壳。

执行规则

Do

  • 保持安静、密集、可预测。
  • 使用真实路径、时间、数量和同步状态。
  • 优先用边框和背景变化建立结构。
  • 尊重桌面与 Android 的导航差异。

Don't

  • 不要在应用里加入营销 hero。
  • 不要使用重阴影、装饰渐变或嵌套卡片。
  • 不要隐藏关键元数据。
  • 不要让 Markdown、代码、路径和文件名横向溢出。
